# Forensic Face & Skin Consistency Workbench

Локальный, воспроизводимый каркас для MacBook M1: **3DDFA_V3 + OpenCV + scikit-image + Skan**. Проект сравнивает техническую согласованность геометрии и текстуры по времени; он не устанавливает личность и не доказывает «двойника».

## Рабочий сценарий: только две команды

```bash
# 1. Same-day калибровка на ваших фотографиях
python run_calibration.py \
  --input /путь/к/моим_фото \
  --output runs/my_calibration

# 2. Основной датасет после успешной held-out проверки калибровки
python run_main_analysis.py \
  --input /путь/к/датасету_1999_2026 \
  --calibration runs/my_calibration \
  --output runs/main_analysis
```

Первый запуск автоматически выполняет Stage 1, pose/zone/Skan-анализ, защиту от утечки near-duplicates, разбиение calibration/validation/test, подбор порогов и held-out проверку false-anomaly rate. Второй запуск отказывается начинать основной анализ, если калибровка не прошла критерий допуска. Подробности: `CALIBRATION_QUICKSTART_RU.md`.

## Ключевой контракт UV

Для каждого снимка создаются два независимых продукта (по умолчанию 768×768, максимум 1000×1000):

1. `uv_analysis.png` — только пиксели, наблюдаемые на исходном фото. Ненаблюдаемые участки чёрные/невалидные; аналитика обязана применять `observed_mask` и `confidence` из `uv.npz`. Цвет, резкость и микротекстура исходных наблюдений не «улучшаются» генеративно.
2. `uv_synthetic.png` — визуальная текстура для 3D/морфинга. Скрытая половина дополняется зеркальным контралатеральным материалом, маленькие внутренние дыры — OpenCV inpaint. Все такие пиксели помечены `synthetic_mask` и исключаются из измерений.

`mesh_analysis.obj` использует аналитическую текстуру; `mesh.obj` — синтетическую. `uv_detail.png` и `uv_ridge_skeleton.png` являются производными диагностическими картами, а не исходными доказательствами.

## Почему это лучше стандартного extractTex 3DDFA_V3

- выборка оригинала идёт прямо в UV по каждому треугольнику с bicubic interpolation;
- учитывается ориентация нормалей и сохраняется непрерывная confidence map;
- аналитические и дорисованные пиксели физически и семантически разделены;
- нет PCA-текстуры в скрытых областях аналитической карты;
- LBP, GLCM, частотные признаки и ridge/skeleton-граф Skan считаются только на наблюдаемой части;
- разрешение ограничено реальной информативностью фото ≤800 px.

## Установка на Apple Silicon

Требования: macOS 13+, Xcode Command Line Tools, Homebrew. Нативный arm64, без Rosetta.

```bash
chmod +x setup_macos_m1.sh
./setup_macos_m1.sh
source .venv/bin/activate
```

3DDFA_V3 inference и bundled Cython renderer запускаются на CPU: MPS не является drop-in заменой nvdiffrast. PyTorch MPS можно использовать позднее для независимых моделей, но Stage 1 намеренно детерминирован на CPU.

## Веса и лицензии

Архив не содержит весов. Поместите официально полученные файлы в `3ddfav3/assets/`:

- `face_model.npy`
- `net_recon.pth` (ResNet-50) или `net_recon_mbnet.pth`
- `large_base_net.pth`

Проверьте условия BFM и каждого набора весов до публикации/распространения.

## Датасет 1999–2026

Имена строго `YYYY_MM_DD[_N].jpg`. Рекомендуемая структура:

```text
dataset/main/
  frontal/
  left_light/ left_mid/ left_deep/ left_profile/
  right_light/ right_mid/ right_deep/ right_profile/
```

Код дополнительно определяет ракурс по yaw. Расхождение папки и рассчитанного bin — QA-флаг, а не автоматическое исправление даты/ракурса.

## Запуск

```bash
python scripts/doctor.py
python app6/run_stage1.py \
  --project-root 3ddfav3 \
  --input dataset/main \
  --output results/stage1 \
  --device cpu --uv-size 768 --limit 10 --fail-fast

python -m unittest discover -s app6/tests -v
```

Сначала gate 10 фото, затем 100, только после этого полный набор. Не смешивайте результаты разных версий кода/config hash.

## Выход одного фото

- 3D: identity/expression meshes, landmarks, pose, visibility;
- UV: `uv_analysis.png`, `uv_synthetic.png`, `uv.npz`, confidence/provenance masks;
- skin: `texture_forensics.json/.npz`, LBP/GLCM/frequency/ridge graph;
- QA: focus/noise/compression proxies, usable skin zones, reprojection checks;
- audit: schema/config/code hashes и атомарная запись результата.

## Pose-aware морщины и Skan

Stage 1 строит 13 канонических зон: центральный/левый/правый лоб, межбровье, обе подглазничные области, обе зоны «гусиных лапок», обе верхние щеки, обе носогубные зоны и подбородок. Для каждого из девяти ракурсов задана отдельная политика: frontal анализирует двусторонние и центральные зоны; light снижает вес дальней стороны; mid/deep/profile оставляют преимущественно наблюдаемую сторону.

Низкое разрешение не скрывается супер-разрешением: система измеряет размер зоны в пикселях исходника, отбрасывает зоны ниже 12 px, оставляет только грубые линии при 12–19 px и повышает доверие после 20/36 px. Для зернистых кадров используются две независимые descriptor-ветки — исходная и консервативно NLM-очищенная. Линия принимается только при согласии обеих веток, устойчивости на нескольких Hessian-масштабах и соответствии мягкому анатомическому orientation prior. Skan получает очищенный skeleton и возвращает ветви, длины и топологию.

Хронологический запуск:

```bash
python scripts/run_wrinkle_chronology.py \
  --stage1 results/stage1 \
  --output results/wrinkle_chronology.json
```

Полный контракт и матрица ракурсов: `WRINKLE_PIPELINE_SPEC.md`.

## Правила вывода

- сравнивать текстуру только в совпадающих ракурсах, зонах и сопоставимом качестве;
- геометрию отделять от выражения, веса, возраста, ракурса и оптики;
- хронологическую аномалию подтверждать несколькими независимыми группами признаков;
- всегда перечислять альтернативы: свет, камера, компрессия, ретушь, макияж, лечение, операции, возраст и вес;
- формулировка результата: «согласовано / частично согласовано / имеются существенные несостыковки», с неопределённостью — не утверждение о подмене личности.
