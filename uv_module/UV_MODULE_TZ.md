## ТЗ: модуль улучшенной UV‑текстуры лица для 3DDFA_V3

### 1. Цель и контекст

- **Цель**: разработать новый модуль генерации UV‑текстуры лица, который:
  - максимально сохраняет детали оригинального изображения (поры, морщины, фактура кожи, контуры глаз/губ/бровей и т.п.);
  - полностью совместим с библиотекой `3ddfa_v3` (топология меша, UV‑layout, форматы вывода);
  - не требует обучения новых моделей (ядро только классический image/geometry processing);
  - корректно и эффективно работает на **MacBook M1** (CPU + `torch.mps` + OpenCV/NumPy, без CUDA);
  - умеет:
    - генерировать HD‑UV‑карту (обычно 1024–2048, опционально до 4096);
    - рендерить OBJ/PLY с этой текстурой;
    - сохранять UV‑карту и дополнительные карты (маски, confidence) в отдельные файлы для анализа.

- **Библиотека**: основная библиотека установлена в `core/3ddfa_v3`. Новый модуль живёт в `uv_module` и интегрируется в пайплайн как **расширение/замена стандартного extractTex**.

---

### 2. Жёсткие требования

- **Совместимость с 3DDFA_V3**:
  - использовать существующую топологию меша (BFM/FFHQ, те же `vertices`, `tri`);
  - использовать существующий UV‑layout (`uv_coords` из BFM/FFHQ);
  - не изменять порядок вершин/граней;
  - форматы вывода:
    - OBJ + MTL + `texture.png` полностью совместимы со стандартным выводом `3ddfa_v3`;
    - модуль можно подключить как drop‑in replacement текущего генератора текстуры.

- **Платформа / окружение**:
  - целевая платформа — **macOS / Apple Silicon (M1)**;
  - допустимые библиотеки: `numpy`, `scipy`, `opencv-python`, `torch` (с `mps` backend), стандартная библиотека Python;
  - **запрещено**: зависеть от CUDA‑библиотек (`nvdiffrast`, PyTorch3D с CUDA‑требованием и т.п.);
  - алгоритмы должны быть реализуемы на CPU / MPS без неадекватного времени работы (ориентир: 1–5 секунд на текстуру 2048²).

- **Без обучения**:
  - ядро модуля не должно требовать обучения/дообучения нейросетей;
  - допускается **опциональный режим** с предобученной face‑restoration моделью (например, GPEN/GFPGAN) как дополнительный шаг «улучшения», но:
    - он должен быть отключаемым флагом;
    - основной пайплайн обязан полноценно работать **без** него.

---

### 3. Входные и выходные данные

#### 3.1. Вход (после стандартного инференса 3DDFA_V3)

Модуль работает **после** стандартной реконструкции `3ddfa_v3`, получая:

- `image: np.ndarray (H, W, 3)` — исходное лицо (BGR/RGB, без даунскейла);
- `recon_dict: dict` / `tddfa_output`, содержащий как минимум:
  - `vertices: np.ndarray (N, 3)` — 3D‑вершины реконструированного лица;
  - `triangles` / `tri: np.ndarray (T, 3)` — индексы треугольников;
  - `uv_coords: np.ndarray (N, 2)` — UV‑координаты в диапазоне `[0, 1]`;
  - `projection_params` / `camera_params` / `pose` — всё, что нужно для проекции 3D→2D;
  - `coeffs` (если доступны):
    - `shape`, `exp` — 3DMM‑параметры формы/выражения;
    - `illumination` (SH‑коэффициенты освещения) — для опционального de‑lighting;
    - `texture` — PCA‑текстура для fallback‑подложки.

Опционально могут передаваться:

- `landmarks` — 2D/3D‑ключевые точки;
- `bfm_symmetry` — карта соответствия вершин left↔right;
- семантические UV‑маски (лоб, глаза, губы, нос, щеки, подбородок и т.п.).

#### 3.2. Выход

Модуль должен уметь возвращать **две версии текстуры** и связанные с ними карты:

- **Текстура для анализа (`uv_tex_analysis`)**:
  - `uv_tex_analysis: np.ndarray (R, R, 3) uint8` — карта, содержащая **только те texel’ы, которые реально были получены из исходного фото**;
  - на ней **не должно быть симметрийно «дорисованных» или синтетически inpaint‑нутых деталей**;
  - невидимые/ненадёжные области должны иметь `uv_mask_visible == 0` и/или нулевой confidence.

- **Текстура для рендера (`uv_tex_beauty`)**:
  - `uv_tex_beauty: np.ndarray (R, R, 3) uint8` — версия для визуализации/рендера;
  - может включать симметрийное заполнение, inpainting, Poisson‑блендинг и, при необходимости, GAN‑refine;
  - строится на основе `uv_tex_analysis`, но отдельно, без изменения исходной «анализируемой» карты.

- **Основные карты**:
  - `uv_mask_visible: np.ndarray (R, R) uint8/bool` — маска texel’ов, где данные взяты напрямую из фото (видимые области);
  - `uv_confidence_map: np.ndarray (R, R) float32` — карта доверия/качества для каждого texel;
  - `uv_is_original: np.ndarray (R, R) uint8/bool` — бинарная карта: 1 — texel с оригинальными данными из изображения (без симметрии/inpaint/GAN), 0 — синтетический/заполненный.

- **Дополнительно (для анализа / расширений)**:
  - `uv_visibility_map: np.ndarray (R, R)` — карта видимости (с учётом z‑буфера/угла к камере);
  - `uv_detail_strength_map: np.ndarray (R, R)` — где и насколько усиливались детали;
  - при включённом de‑lighting:
    - `shading_uv: np.ndarray (R, R, 3)` — оценка освещённости;
    - `albedo_uv: np.ndarray (R, R, 3)` — приблизительная карта альбедо.

- **Для рендера / экспорта**:
  - структуру `obj_data`, содержащую:
    - вершины, треугольники, uv‑координаты;
    - имя/путь до текстуры;
  - сохранение:
    - `{base}.obj`, `{base}.mtl`, `{base}_texture.png`;
    - `{base}_uv.png` — визуализация UV‑карты;
    - `{base}_confidence.png` / `{base}_confidence.exr` — карта качества.

### 4. Общая архитектура модуля

Модуль проектируется модульно, с основным классом и вспомогательными компонентами.

#### 4.1. Главный класс

```python
class HDUVTextureGenerator:
    def __init__(
        self,
        uv_size: int = 2048,
        super_sample: int = 2,
        enable_delighting: bool = True,
        enable_symmetry_fill: bool = True,
        enable_detail_boost: bool = True,
        use_fast_path: bool = True,
        device: str = "mps",  # 'mps' или 'cpu' на MacBook M1
    ):
        ...

    def generate(self, image: np.ndarray, recon_dict: dict):
        """
        Выполняет полный пайплайн генерации HD‑UV‑карты.

        Возвращает:
            uv_tex_analysis      # текстура ТОЛЬКО из реально видимых пикселей фото
            uv_tex_beauty        # текстура для рендера (симметрия + inpaint + блендинг/GAN по флагу)
            uv_mask_visible      # маска texel'ов, где цвет взят из фото напрямую
            uv_confidence_map    # карта доверия к каждому texel'у
            aux_data             # shading_uv, albedo_uv, uv_visibility_map и др. вспомогательные карты
        """
        ...
```

#### 4.2. Вспомогательные компоненты

Разделить реализацию на подмодули (можно как отдельные классы/файлы в `uv_module`):

- `VisibilityRenderer` / `UVVisibility` — расчёт видимости, глубины и весов:
  - пер‑треугольный z‑buffer;
  - вычисление нормалей, угла к камере, видимости (front‑/back‑face);
  - выдаёт per‑triangle и/или per‑texel `visibility_weight`.

- `UVBaker` / `UVProjector` / `UVSuperSampler` — основной UV‑baking:
  - high‑res UV (целевой `uv_size`) + внутренний `work_size = uv_size * super_sample`;
  - per‑треугольный **аффинный варпинг** из image→UV (основа из идей `hd_uv_texture.py`):
    - для каждого треугольника строится `M: UV_triangle → image_triangle`;
    - rasterization bounding box в UV‑пространстве;
    - для каждого texel в треугольнике:
      - вычислить координаты в исходном изображении;
      - взять цвет `cv2.remap`/INTER_CUBIC;
      - аккумулировать цвет в `color_accum`, вес — в `weight_accum` с учётом видимости/угла;
  - опционально — дополнительно UV‑grid‑based super‑sampling (UV→image, как в `UVSuperSampler` / `HiFiUVExtractor`) для особенно тонких деталей.

- `DeLighter` (опционально) — нормализация освещения:
  - использует SH‑коэффициенты освещения и нормали для оценки `shading_uv`;
  - строит `albedo_uv ≈ texture / shading_uv` с мягким clamping (чтобы не «выжечь» тени).

- `UVInpainter` — закрытие дыр и сглаживание:
  - симметрина (mirror)‑дозапись невидимых зон по UV (и/или по `bfm_symmetry`):
    - копирование цвета с противоположной стороны;
    - локальная color‑correction (histogram/mean–std в окне);
    - мягкое смешивание на границе;
  - классический `cv2.inpaint` (Telea/NS) для остаточных дыр;
  - gradient‑domain / Poisson‑подобный блендинг швов (упрощённый Poisson / Laplacian smoothing по seam‑зонам).

- `UVDetailEnhancer` — работа с base/detail:
  - перевод в удобное цветовое пространство (Lab/YCrCb);
  - разделение на:
    - **base layer** — low‑frequency составляющая;
    - **detail layer** — high‑frequency остаток;
  - селективное усиление detail‑слоя:
    - использование семантических UV‑масок (глаза, губы, нос и т.п., если доступны);
    - усиление деталей в «важных» зонах (глаза/губы/нос/брови);
    - ослабление усиления в низкоконфидентных/окклюзированных областях;
  - варианты реализации:
    - multi‑band/Laplacian pyramid;
    - unsharp mask над detail;
    - локальный Laplacian filter + boosting.

- `UVIO` — сохранение и интеграция:
  - функции записи OBJ/MTL/PNG/EXR, совместимых с `3ddfa_v3`;
  - утилиты:
    - сохранение отдельной UV‑карты;
    - сохранение масок/карт качества;
    - подготовка структуры `obj_data` для внешних рендеров.

- **(опционально)** `UVRefinerGAN`:
  - обёртка над предобученным GPEN/GFPGAN;
  - режим «GAN‑refine» включается флагом;
  - на вход берёт исходное фото и/или базовую UV;
  - на выход отдаёт «очищенную/улучшенную» версию, которая сливается с классической текстурой с учётом масок качества.

### 5. Детальный пайплайн (по шагам)

#### 5.1. Подготовка

1. Принять исходное изображение **в максимальном доступном разрешении**:
   - если внутри `3ddfa_v3` использовался `--iscrop`, выполнить обратный affine‑трансформ, чтобы работать в координатах оригинального кадра.
2. Задать:
   - `uv_size` (целевое разрешение UV);
   - `super_sample` (`1–4`), определяющий внутренний `work_size = uv_size * super_sample`.

#### 5.2. 3D → 2D проекция и геометрия

1. Использовать `vertices`/`tri` из `recon_dict` (либо пересобрать из `coeffs` при необходимости).
2. Получить:
   - `vertices_3d (N, 3)`;
   - `vertices_2d (N, 2)` — проекция на исходное изображение (по правилам `3ddfa_v3`).
3. Не менять топологию:
   - `tri` и `uv_coords` должны оставаться согласованными с BFM/FFHQ.

#### 5.3. Видимость и веса качества

1. По треугольникам:
   - вычислить нормали (в 3D; можно через cross‑product по вершинам);
   - определить front‑face/back‑face по косинусу угла с направлением взгляда;
   - опционально использовать z‑buffer/visibility pass для более точного отсечения.
2. Для каждого треугольника вычислить **вес видимости/качества**:
   - `w_vis` — видимость (0/1 или мягкая);
   - `w_angle` — штраф за большой угол к камере (например, `cos(theta)^gamma`, игнорировать >80–85°);
   - при желании учесть:
     - положение в кадре (центр vs край);
     - размер проекции (очень маленькие треугольники можно ослабить);
     - локальную резкость в image (по Laplacian/gradient).
3. Объединить в итоговый per‑triangle вес `w_tri ∈ [0,1]`, сохраняемый для UV‑baking.

#### 5.4. High‑Resolution UV‑baking

1. Преобразовать `uv_coords` в пиксели рабочей UV‑карты:
   - `u_px = u * (work_size - 1)`;
   - `v_px = (1 - v) * (work_size - 1)` (если V направлен сверху вниз).
2. Для каждого треугольника с ненулевым весом `w_tri`:
   - взять:
     - `src_pts = verts_2d[tri]` — треугольник в пространстве изображения;
     - `dst_pts = uv_px[tri]` — треугольник в UV‑пикселях;
   - проверить, что треугольники не вырождены (площадь > порога);
   - посчитать матрицу `M = cv2.getAffineTransform(dst_pts, src_pts)` (UV → image);
   - вычислить bounding box треугольника в UV‑пространстве;
   - внутри bounding box:
     - сгенерировать маску треугольника;
     - для каждой точки маски:
       - вычислить image‑координаты через `M`;
       - проверить, что точка попадает внутрь исходного изображения;
       - взять цвет из `image` через `cv2.remap`/INTER_CUBIC;
       - записать в `color_accum[y, x] += color * w_tri`;
       - записать вес в `weight_accum[y, x] += w_tri`.
3. После прохода по всем треугольникам:
   - нормализовать:
     - `uv_texture[y, x] = color_accum[y, x] / weight_accum[y, x]` там, где вес > 0;
   - сформировать:
     - `uv_mask_visible = (weight_accum > 0)`;
     - начальный `uv_confidence_map` на основе нормированных весов;
     - начальную бинарную карту `uv_is_original = uv_mask_visible.copy()`.

#### 5.5. Base + Detail разложение и усиление деталей

1. Сформировать «сырой» вариант текстуры для анализа:
   - `uv_tex_analysis_raw = uv_texture.copy()`;
   - `uv_mask_visible` и `uv_confidence_map` считаются основой именно для **аналитической** версии.
2. Перевести `uv_tex_analysis_raw` в цветовое пространство, где яркость отделена (`Lab`/`YCrCb`).
3. Получить **base layer** (низкие частоты):
   - через bilateral/guided/edge‑preserving фильтр;
4. Получить **detail layer**:
   - `detail = original - base` (либо разность в лог‑яркости).
5. Сформировать **карту усиления деталей**:
   - использовать:
     - `uv_confidence_map` (где сильные детали, где нет данных);
     - семантические маски (глаза/губы/нос/брови/лоб/щеки/подбородок);
   - задать:
     - более высокий коэффициент усиления в важных зонах (глаза, губы, нос, брови);
     - ослабленный коэффициент в слабонаблюдаемых/шумных областях.
6. Усилить детали **для аналитической версии**:
   - `uv_tex_analysis = base + gain * detail` (с ограничением по диапазону);
   - важно: никакие симметрийные или inpaint‑операции к `uv_tex_analysis` не применяются.

#### 5.6. De‑lighting (опционально)

1. При наличии SH‑коэффициентов освещения:
   - по нормалям и SH‑параметрам оценить `shading_uv` (гладкая карта освещённости);
2. Выполнить приближённое разделение `texture ≈ albedo * shading`:
   - `albedo_uv = uv_texture_enhanced / max(shading_uv, epsilon)` с clamping;
3. Решить на уровне конфигурации:
   - выдаём ли пользователю именно albedo или слегка нормализованную текстуру (частично сохраняя shading).

#### 5.7. Симметрийное заполнение невидимых областей

1. По `uv_confidence_map`/`weight_accum` определить слабонаблюдаемые области (низкий вес).
2. Найти симметричные texel’ы:
   - упрощённо — по зеркалу относительно `u ≈ 0.5`;
   - при наличии `bfm_symmetry` — по точной карте соответствия вершин.
3. Копировать базовый цвет/детали с симметричной стороны:
   - перед копированием выполнить локальную color‑correction (сопоставление средних/дисперсий в небольшом окне);
4. Смешать симметричную информацию с исходной:
   - `UV_final = c_obs * UV_obs + c_sym * T(UV_sym)` с мягким переходом;
   - границу симметричного заполнения размыть (Gaussian / multi‑band) для избежания видимого шва.

#### 5.8. Inpainting остаточных дыр

1. Определить пиксели, где после симметрии всё ещё нет данных (`uv_mask == 0` / низкий confidence):
2. Использовать:
   - `cv2.inpaint` (Telea или Navier‑Stokes) в UV‑пространстве;
   - по возможности:
     - для base‑слоя — edge‑aware diffusion;
     - для detail‑слоя — patch‑based synthesis из донорских регионов (аналог PatchMatch, но можно упрощённо, без тяжёлых зависимостей).

#### 5.9. Gradient‑domain / Poisson‑подобный блендинг швов

1. Найти зоны швов:
   - по градиенту `uv_confidence_map`/`weight_accum` (резкие переходы);
   - по маске симметричного заполнения и inpaint‑зон.
2. Выполнить облегчённый Poisson‑/Laplacian‑блендинг:
   - итеративный Laplacian smoothing только в seam‑зоне с якорением внутренних пикселей;
   - либо использование `cv2.seamlessClone` на small‑patch уровнях.

#### 5.10. Финальный detail‑boost и даунсэмплинг

1. Применить финальный мягкий unsharp mask / локальный detail‑boost:
   - усиливать только там, где `uv_confidence` достаточно высок;
   - не перешарпливать inpaint‑зоны.
2. Если `super_sample > 1`:
   - downsample с `INTER_AREA` с `work_size → uv_size`;
   - это сгладит aliasing, сохраняя уже «впекшиеся» детали.

#### 5.11. Экспорт / интеграция

1. Сохранить:
   - `uv_texture` как `{base}_texture.png`;
   - OBJ/MTL в формате, который ожидает `3ddfa_v3`:
     - `map_Kd {base}_texture.png` в MTL;
2. Сохранить вспомогательные карты (по конфигурации):
   - `{base}_uv.png`;
   - `{base}_confidence.png`;
   - при желании EXR/TIFF‑форматы для float‑карт.

### 6. Опциональный GAN‑режим (UVRefinerGAN)

Отдельным модулем (по флагу конфигурации) допускается использование предобученной GAN‑модели (GPEN/GFPGAN) как «усилителя»:

- вход: оригинальное фото и/или базовая HD‑UV‑текстура;
- внутри:
  - GAN улучшает локальное качество/детали лица;
  - результат проецируется обратно в UV‑пространство с учётом геометрии;
- выход:
  - улучшенная карта, которая смешивается с базовой HD‑UV по маскам качества/регионов.
